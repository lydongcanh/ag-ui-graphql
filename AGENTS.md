# Coding Instructions

- Always ask clarifying questions before answering to ensure you understand the user's request.
- Always follow best coding practices, including writing clean, maintainable code, single responsibility principle, and proper encapsulation.
- Never implement a workaround or hack, always implement the correct solution to the problem.
- When implementing a new feature, consider how it will fit into the existing architecture and design of the codebase. If necessary, refactor existing code to accommodate the new feature in a clean and maintainable way.
- Each module only has 1 manager class that can attach to the node of the screen. All other classes are just pure C# classes that can be instantiated and used by the manager class.
- Comment your code smartly like a true senior developer, explaining the purpose of each class, and important logic. This will help other developers (or your future self) understand the code quickly.
- 1 file = 1 class, no exceptions. If a class is too large, break it down into smaller classes. This promotes separation of concerns and makes the code easier to maintain.
